Publicar asset

Configura y publica el asset antes de definir política y contrato.

La imagen se sube al servicio local del conector y se publica automaticamente en el metadata del asset.

Se sube al servicio local del conector y luego se publica el asset usando esa URL interna.

El FeatureLayer se exportará en el formato elegido cada vez que se transfiera el asset. Usa Access token (login ArcGIS) si la capa requiere autenticación.

Autenticación opcional

ArcGIS pendiente de login

Si el origen del asset está en ArcGIS Enterprise, selecciona Access token (login ArcGIS). La UI reutiliza la sesión del portal y solicita el token necesario para publicar o transferir.

Mini tutorial

  1. Introduce `Asset key` y nombre del asset.
  2. En `Origen del asset`, elige `Base URL remota`, `Archivo local` o `ArcGIS FeatureLayer`.
  3. Si eliges `Base URL remota`, completa `Base URL` y `Path`, y si necesitas autenticacion de origen, selecciona tipo y secreto del vault.
  4. Si eliges `Archivo local`, selecciona el fichero desde tu equipo. La UI lo subirá al servicio local y publicará el asset con esa fuente.
  5. Pulsa "Crear/Actualizar asset" y confirma en consola que devuelve `2xx`.

Definir política de uso

Define las condiciones de uso del dato según estándares ODRL / DCAT. El formulario genera la política automáticamente.

Política de acceso dct:accessRights
Finalidad de uso dct:purpose
Tiempo de acceso odrl:dateTime
Prohibiciones odrl:prohibition
Obligaciones odrl:duty

Política de uso ODRL/DCAT

  1. Elige la política de acceso: público, privado, socios o interno.
  2. Selecciona la finalidad principal con la que se usarán los datos.
  3. Define el tiempo de acceso o deja "Siempre" para acceso indefinido.
  4. Marca las prohibiciones que apliquen (redistribución, reidentificación).
  5. Marca las obligaciones si el consumidor debe citar la fuente o permitir auditoría.
  6. Pulsa Crear/Actualizar policy y continúa a Contrato.

La política se genera como ODRL Set con permisos, prohibiciones y obligaciones conforme a DCAT-AP y GAIA-X.

Solicitudes de acceso recibidas

Solicitudes enviadas por otros conectores para acceder a tus assets privados.

Asset Solicitante Organización Finalidad Fecha Estado Acciones
Pulsa "Actualizar" para cargar las solicitudes.

Cómo funciona

  1. Cuando publicas un asset con visibilidad Privado, los consumidores solo pueden solicitar acceso, no contratar directamente.
  2. La solicitud llega aquí y, si tienes SMTP configurado, también recibes un email en el correo del dueño del asset.
  3. Revisa la finalidad y aprueba o rechaza la solicitud. El consumidor puede consultar el estado.

Transferencias

Si dejas campos vacíos, la UI autocompleta metadatos (titulo, tags, descripcion y snippet) para que el item no se publique sin contexto.

TransferenciaEstadoContrato

Mini tutorial

  1. Selecciona contrato desde el desplegable o pega el `agreementId` real.
  2. En `Modo de salida`, elige `Enviar a URL destino` o `Descargar en local`.
  3. Si eliges `Enviar a URL destino`, completa `Destination URL` del receptor en produccion.
  4. Si eliges `Descargar en local`, el navegador descargará el recurso directamente y lo guardará según tu configuración, normalmente en Descargas.
  5. Si la transferencia remota se queda en STARTED, pulsa Dataplanes para verificar que hay un motor activo.
  6. Si sigue atascada, pulsa Terminar en la fila y republica el asset con el token renovado antes de reintentar.

Proyecto EITEL

            
EITEL impulsa un espacio de datos energéticos para gobiernos locales, con enfoque en soberanía del dato, confianza y colaboración entre municipios.

            

              

                
Qué es EITEL

                
Un espacio de datos para compartir información energética bajo términos propios, alineado con GAIA-X y orientado al beneficio ciudadano.

              

              

                
Impacto

                
Más de 700K ciudadanos beneficiados, 4 municipios adheridos y 6 socios en la iniciativa, según el portal oficial.
